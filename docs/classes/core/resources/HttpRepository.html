    Instance Method Details

    - (String) resource_url(collection_name, id, includes, action)

    Generates the URL to access a RESTful collection or resource.

    Examples:

    a collection of all users: 

    repository.resource_url( "users" ) # "/api/users/"

    URL for a specific user: 

    repository.resource_url( "users", 1 ) # "/api/users/1"

    URL with a specific action: 

    repository.resource_url( "users", 1, null, "edit" ) # "/api/users/1/edit"

    follows the recent_posts association (note: dependent on the implementation of gen_req_params): 

    repository.resource_url( "users", 1, { recent_posts: true } ) # "/api/users/1?include=recent_posts"

    Parameters:

    • collection_name (String) — the name of the collection.
    • id (String) — (optional) the id of the resource.
    • includes (Object) — (optional) the associations to follow and include in the response.
    • action (String) — (optional) the action to append to the collection URL.

    Returns:

    • (String) — the full URL.

    - (String) gen_req_params(associations)

    Generates query params to follow the given associations on the server and include them in the response.

    Examples:

    repository.gen_req_params( user, blogs: { blog_posts: true } ) # "include=user,blogs.blog_posts"

    Parameters:

    • associations (Object) — a tree representing the associations.

    Returns:

    • (String) — the query param.
